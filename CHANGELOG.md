# Changelog

## [0.1.0] — 2026-06-02

### Added
- **4 new theme variants:** Black Night, Black Mocha, Black Neon, Black Minimal
- **README** with variant overview, installation guide, and feature list
- Border color per variant now matches the variant's tint (Night: navy, Mocha: brown, Neon: dark green, Minimal: near-black)

### Changed
- Extension renamed to **Black Theme**
- Version bumped to `0.1.0`
- Improved border consistency — all panels, sidebars, tab bars, terminals, and widgets now have matching visible borders across all variants

### Variants (complete list)
- Black Bordered *(original)*
- Black Bordered Lite
- Black Bordered High Contrast
- Black Bordered Warm
- Black Night *(new)*
- Black Mocha *(new)*
- Black Neon *(new)*
- Black Minimal *(new)*

## [0.0.2] — 2026-05-22

### Added
- Initial release with 4 variants: Black Bordered, Black Bordered Lite, Black Bordered High Contrast, Black Bordered Warm
- Visible borders on all panels and UI elements
- Full semantic token support

## [0.2.0] — 2026-06-02

### Changed
- Completely reworked syntax highlighting across all 8 variants — much more colorful
- HTML tags, attributes, IDs, classes now have distinct colors
- CSS selectors, pseudo-classes, units, colors, variables, at-rules all individually colored
- JSON keys, values, brackets, booleans each get their own color
- Markdown H1/H2/H3 now have individual colors
- Added Regex group, character class, and quantifier coloring
- Added Shell/Bash command, variable, and shebang coloring
- Added Rust lifetime, macro, attribute, and unsafe coloring
- Brackets/braces now colored gold across all themes
- Each variant's colorful palette matches its tint (Night=blues, Mocha=ambers, Neon=max saturation, Minimal=desaturated)
